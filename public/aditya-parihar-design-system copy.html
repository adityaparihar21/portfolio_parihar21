05 Design Principles

// AP_SYS — core.design.principles

Micro-animations over static UI
The UI should feel alive. Always prefer subtle hover states over static cards. Motion implies intelligence.
Dual identity architecture
Cold precision for the technical world. Warm analog warmth for the creative world. Never mix the two mid-section.
Terminal as personality
The AP_SYS terminal responses feel like an authentic UNIX environment — factual, slightly cold, with hidden easter eggs.
Gold as the constant
The AP monogram in #d4af37 is the only element that appears unchanged across both the System and Cinematic worlds.
06 Tone of Voice

Brand Manifesto

"Most code solves a problem. Good code tells a story. Great code doesn't need to explain itself."

◈
Confident & Direct
Avoid filler words. Use short, punchy sentences. Every word earns its place.
◉
Technical yet Cinematic
Describe systems as architecture and logic, but frame output as experience and narrative.
▣
CLI Persona
AP_SYS feels like a real UNIX environment — authentic, cold, with hidden easter eggs.
